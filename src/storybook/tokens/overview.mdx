## Token Architecture

<Box p="6" border="1px solid" borderColor="border-default" borderRadius="lg" mt="8">
  <Heading size="md" mb="4">
    How Tokens Work
  </Heading>

  <VStack gap="4" alignItems="flex-start">
    <Box>
      <Heading size="sm" mb="2">
        üîß Primitive Tokens
      </Heading>
      <Text fontSize="sm">
        Raw design values defined in <code>src/styles/primitives/</code>. These are the foundation and don't change based on theme.
      </Text>
    </Box>

    <Box>
      <Heading size="sm" mb="2">
        üé≠ Semantic Tokens
      </Heading>
      <Text fontSize="sm">
        Theme-aware tokens defined in <code>src/styles/semantics/</code>. These reference primitive tokens and automatically switch values for light/dark themes.
      </Text>
    </Box>

    <Box>
      <Heading size="sm" mb="2">
        üîÑ Automatic Synchronization
      </Heading>
      <Text fontSize="sm">
        All stories import tokens directly from source files, ensuring visualizations stay in sync with code changes.
      </Text>
    </Box>

  </VStack>
</Box>

## Usage Guidelines

<Box p="6" border="1px solid" borderColor="border-default" borderRadius="lg" mt="8">
  <Heading size="md" mb="4">
    Best Practices
  </Heading>

  <VStack gap="3" alignItems="flex-start">
    <Text fontSize="sm">
      ‚úÖ <strong>Use semantic tokens:</strong> Prefer <code>bg.surface.default</code> over raw color values for theme compatibility
    </Text>
    <Text fontSize="sm">
      ‚úÖ <strong>Follow spacing scale:</strong> Use predefined spacing values (4, 8, 16, 24, etc.) instead of arbitrary values
    </Text>
    <Text fontSize="sm">
      ‚úÖ <strong>Choose appropriate text styles:</strong> Use <code>textStyle.*</code> tokens for consistent typography hierarchy
    </Text>
    <Text fontSize="sm">
      ‚úÖ <strong>Test in both themes:</strong> Semantic tokens automatically adapt, but verify the result looks good in light and dark modes
    </Text>
  </VStack>
</Box>
